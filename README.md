# 台股強勢股隔日放空策略研究

## 研究摘要

本研究探討「**用台股大盤盤面型態（region）篩選日子，隔日放空當日強勢股**」的當沖策略。
從「大盤走勢分類 → 強勢股定義 → 多指數共振 → 市值／周轉率濾網 → 當沖限制 →
分點籌碼 → 停損優化」逐步收斂，最終得到一個在 2015–2026、**含停損與滑價成本**下
仍有正期望、逐年 10/12 年為正、且不依賴單一年份的策略：

> **當台指期、加權指數、櫃買指數「同日都開低走高」時，隔日開盤放空當日漲幅 8–9.5%
> 的小型（市值≤100億）、高周轉（≥0.5%）強勢股，收盤回補。**

研究過程中也排除了大量「看起來漂亮但其實是單日樣本假象」的組合（漲停事件群聚、
TX×S&P500 交叉等），最終結論建立在數百個不同交易日的樣本上。

## 最終策略

| 參數 | 值 |
|------|----|
| 訊號 | TX **AND** TAIEX **AND** TPEx 同日「開低走高」 |
| 標的 | 當日漲幅 8% ~ 9.5%（排除接近漲停） |
| 市值上限 | ≤ 100 億（10,000,000,000） |
| 周轉率下限 | ≥ 0.5% |
| 當沖資格 | 可現股當沖賣（排除暫停先賣後買名單） |
| 進場 | 隔日開盤放空 |
| 出場 | 隔日收盤回補 |
| 停損 | 8% |
| 滑價 | 單邊 0.15% |
| 手續費 | 單邊 0.001425 × 0.6（六折），買賣皆收 |
| 當沖交易稅 | 0.003 × 0.2（二折），賣出收 |
| 股數 | 1000（一張） |

（所有參數定義於 `index_region_backtest.py` 最上方常數區。）

## 研究流程

### Step 1：大盤走勢分類
把每個交易日依「開盤 vs 前日收盤 × 收盤 vs 開盤」分成四種 region：
開高走高 / 開高走低 / **開低走高** / 開低走低。先在台指期上發現「開低走高日 → 隔日放空
強勢股」是四個 region 中唯一穩定為正、且跨三個指數一致的方向。
→ `output/signal_combo.png`

### Step 2：強勢股定義
標的取「當日大漲、但未鎖漲停」的股票（漲幅 8–9.5%，排除 ≥9.5% 接近漲停者，
因為漲停是強制停板、不代表真實賣壓可放空）。以 TPEx 開低走高 × 8% 為例，逐年雖有
波動，但已具雛形。
→ `output/tpex_kdzg_8_yearly.png`

### Step 3：多指數共振
要求 TX、TAIEX、TPEx **三個指數同日都開低走高**（AND）。隨著要求的指數越多，Sharpe、
勝率、回撤都同步改善（單一→雙重→三重），且樣本仍有上百個交易日——這是「加條件帶來
真實改善」而非碎片化假象。
→ `output/signal_combo.png`

### Step 4：市值濾網
把強勢股依市值分組回測，發現**大型股的 +8% 是動能會續漲（放空被軋、巨額回撤）**，
**小型股的 +8% 才容易隔日回落**。限制市值 ≤ 100 億後，Sharpe 大幅提升、最大回撤縮一個量級。
→ `output/size_grid.png`

### Step 5：周轉率濾網
同方向發現：**低周轉的強勢股是淨輸家**，周轉率越高越好。加上周轉率 ≥ 0.5% 濾網，
進一步去掉品質差的樣本。
→ `output/turnover_grid.png`

### Step 6：當沖限制分類
把標的分成「可現股當沖賣 / 暫停先賣後買 / 可資券沖」三類。約八成事件同時具備融券額度
（可資券沖），且該子集績效甚至略優——部分緩解「最難放空的小型飆股借不到券」的疑慮，
但「有額度」不等於「借得到券、借券費可接受」，仍是實單最大風險。
→ `output/daytrade_class.png`

### Step 7：分點主力集中度
用分點資料計算「前 3 大淨買超分點 ÷ 當日成交量」的主力集中度。結果與直覺相反：
**散戶分散買（低集中）的強勢股隔日放空勝率最高（~72%）**，主力集中反而較難放空。
（分點資料僅 2022 起，樣本受限。）此維度有區分力但未納入最終濾網。
→ `output/broker_concentration.png`

### Step 8：停損優化
掃描 1%~10% 與不設停損，發現**盤中緊停損會把「盤中衝高、收盤回落」的均值回歸單洗在最高點**，
停損越緊績效越差。8–10% 寬停損幾乎不觸發、對績效無損，又能封住「跑掉鎖漲停」的災難尾部，
故採 8%。
→ `output/stoploss_grid.png`

### Step 9：最終結果
套用以上所有濾網與參數，得到最終策略的資金曲線與逐年表現：總損益 +188,775、勝率 58.6%、
Sharpe 3.16、10/12 年為正且不依賴單一年份。
→ `output/final_equity_curve.png`、`output/final_yearly.png`

## 檔案結構

```
project/
├── README.md                  # 本檔
├── main_final.py              # ★ 最終策略完整回測（摘要+逐年+資金曲線）
├── index_region_backtest.py   # 研究主引擎：region 分類、強勢股、門檻/市值/周轉率/停損 grid、分組分析
├── main.py                    # 早期主流程（全市場漲停股 × TX region 維度回測）

├── portfolio_backtest.py      # PortfolioBacktester（投資組合回測引擎 + optimize）
├── data_loader.py             # 資料載入（日線/期貨/指數/市值/漲跌停/暫停當沖 + 本地快取）
├── market_scanner.py          # 漲停/強勢股事件偵測、當沖可放空過濾
├── strategy.py                # region 分類與訊號建立
│
├── test_size_turnover.py      # 市值/周轉率資料源探測
├── test_broker.py             # 分點/法人資料源探測 + 鎖漲停分點計算
│
├── data/                      # 個股/指數/市值/分點 CSV 快取（自動建立）
├── output/                    # 所有圖表（含 final_*.png）
└── archive/                   # 已放棄/實驗性腳本（見 archive/README.md）
```

## 執行方式

```bash
# 設定 FinMind token
cp .env.example .env   # 編輯填入 FINMIND_TOKEN

# 跑最終策略（個股資料 cache-only；首次需先建立 data/ 快取）
python main_final.py
```

> 最終策略採 cache-only 讀取 `data/*.csv`，僅指數/市值等需少量 API。完整研究流程
> （各 grid 與分組分析）見 `index_region_backtest.py`。
